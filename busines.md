## 2. Обзор продукта

**ForzaFit** — кроссплатформенный трекер тренировок и питания (Web → iOS → Android). Пользователь выбирает режим: только тренировки, только питание или полный фитнес-трекер. Это снижает порог входа и позволяет монетизировать разные сегменты независимо.

### 1.1 Ключевые цели
- Записывать силовые тренировки (веса, подходы, повторения) за ≤ 3 секунды
- Отслеживать питание (КБЖУ, приёмы пищи, продукты)
- Визуализировать прогресс: графики, рекорды, динамика веса
- Монетизация через Premium-подписку по каждому режиму
- Работать на iOS, Android и в браузере с единым аккаунтом

### 1.2 Ключевые метрики успеха

| Метрика | Цель (6 мес. после запуска) |
|---------|----------------------------|
| DAU / MAU | ≥ 20% |
| Retention D7 / D30 | ≥ 35% / ≥ 15% |
| Среднее время сессии | ≥ 3 минуты |
| Конверсия в Premium | ≥ 3% от установок |
| Рейтинг App Store / Google Play | ≥ 4.5 ★ |
| Доля Web-пользователей от MAU | ≥ 15% через 6 мес. |

## 3. Режим работы приложения

При первом запуске пользователь выбирает режим. Интерфейс, навигация и набор функций адаптируются. Режим можно изменить в настройках.

| Режим | Что включено | Скрыто |
|-------|-------------|--------|
| 🏋️ Тренировки | Дневник тренировок, база упражнений, программы, таймер отдыха, прогресс, трекер тела | Модуль питания |
| 🥗 Питание | Дневник КБЖУ, счётчик калорий, сканер штрих-кода, трекер веса | Модуль тренировок |
| ⚡ Всё сразу | Полный функционал обоих режимов | Ничего |

## 7. iOS-приложение (следующий шаг)

### 7.1 Стек

| Слой | Решение | Обоснование |
|------|---------|------------|
| Framework | React Native + Expo | Переиспользование логики с Web, быстрая разработка |
| Навигация | Expo Router (file-based) | Аналог Next.js App Router, знакомая структура |
| UI Kit | NativeWind (Tailwind для RN) + собственные компоненты | Схожий стиль с Web-версией |
| Стейт | Zustand + TanStack Query | Тот же стек, что и на Web |
| Auth | Firebase Auth (React Native Firebase) | Единый аккаунт с Web |
| HTTP | Axios + тот же interceptor | Переиспользовать логику токенов |
| Хранилище | MMKV (замена AsyncStorage) + SQLite для офлайн | Быстрее AsyncStorage в 10x |
| Push | Expo Notifications | iOS + Android одновременно |
| Сканер | expo-camera + expo-barcode-scanner | Для режима «Питание» |

## 12. Риски и ограничения

| Риск | Влияние | Митигация |
|------|---------|----------|
| Высокая конкуренция (Hevy, Strong, MyFitnessPal) | Высокое | Фокус на UX, быстрый ввод данных, русскоязычная аудитория |
| Низкий Retention без формирования привычки | Высокое | Streak, push-уведомления, расписание тренировок |
| Сложность поддержки трёх платформ | Среднее | Максимальный переиспользуемый код (React/RN + общий backend) |
| FatSecret API: зависимость от провайдера | Среднее | Собственный кеш + возможность смены провайдера |
| GDPR / 152-ФЗ при хранении биометрических данных | Среднее | AES-256 шифрование, локальное хранение, консультация юриста |
| App Store Review (подписка, данные здоровья) | Низкое | Соблюдение гайдлайнов с первой итерации |
